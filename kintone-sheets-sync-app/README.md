# kintone / Google Sheets Sync Console

kintone と Google スプレッドシートの同期アクションを、スプレッドシートの `設定` シートから選択して実行するWebアプリです。

公開URL: https://g.it-mei.com/kintone

このCodex環境からは対象スプレッドシートが `403 PERMISSION_DENIED` で読めなかったため、実シートの列名を推測で固定せず、ヘッダー名で自動認識する実装にしています。`APIトークン` 系の列は画面に返しません。

## ファイル構成

| パス | 内容 |
| --- | --- |
| `index.html` / `styles.css` / `app.js` | ローカル確認用の静的プロトタイプ |
| `gas/Code.gs` | Apps Script サーバー処理。設定読込、kintone API呼び出し、同期ログ作成 |
| `gas/Index.html` / `gas/Styles.html` / `gas/Client.html` | Apps Script Webアプリ画面 |
| `gas/appsscript.json` | Apps Script の権限設定 |

## 設定シートの推奨列

| 列名 | 必須 | 内容 |
| --- | --- | --- |
| `アクション名` | 必須 | 画面のプルダウンに表示する名前 |
| `kintoneドメイン` | 必須 | `example.cybozu.com` または `https://example.cybozu.com` |
| `アプリID` | 必須 | kintoneアプリID |
| `APIトークン` | 必須 | kintone APIトークン。画面には `保存済み（非表示）` とだけ表示 |
| `対象シート` | 必須 | 同期先/同期元のスプレッドシート内タブ名 |
| `キー項目` | 推奨 | upsertに使うkintoneフィールドコード。重複禁止の文字列または数値フィールド推奨 |
| `項目マッピング` | 推奨 | `スプシ列:kintoneフィールドコード` をカンマまたは改行区切りで記載 |
| `kintone⇨スプシ` | 必須 | `◯` / `○` / `〇` / `true` / `1` / `有効` ならボタンを有効化 |
| `スプシ⇨kintone` | 必須 | 同上 |
| `取得条件` | 任意 | kintone query。未指定時は `limit 500` |
| `同期上限` | 任意 | 最大500件まで |
| `有効` | 任意 | 空欄なら有効、`◯` 等なら有効、それ以外は無効 |
| `ゲストスペースID` | 任意 | ゲストスペース内アプリの場合のみ |

マッピング例:

```text
管理ID:management_id, 会社名:company_name, 担当者:owner, ステータス:status
```

## Apps Script での設置

1. 対象スプレッドシートを開く
2. `拡張機能 > Apps Script` を開く
3. `gas/` 配下の同名ファイルを Apps Script プロジェクトへ追加する
4. `Code.gs` の `APP_CONFIG.spreadsheetId` が対象スプレッドシートIDになっていることを確認する
5. `デプロイ > 新しいデプロイ > ウェブアプリ` で公開する
6. 初回は `テスト` モードで件数確認だけ行い、ログを確認してから `本実行` に切り替える

## 現MVPの同期仕様

- `kintone⇨スプシ`: kintoneの複数レコードを取得し、対象シートへ一意キーで更新または追加する
- `スプシ⇨kintone`: 対象シートの行をkintoneへ送信する。`キー項目` がある場合は複数レコード更新APIの `upsert` を使う
- `テスト` モード: kintone取得件数または送信予定件数だけ確認する
- `本実行` モード: 実データを書き込み、`同期ログ` シートへ結果を保存する
- 対象シートには必要に応じて `kintoneレコードID`、`kintoneリビジョン`、`同期ステータス`、`同期エラー`、`最終同期日時` を追加する
- 削除同期は入れていない

## 追加すると良い仕様

- 差分プレビュー: 本実行前に「追加」「更新」「スキップ」「エラー予定」を件数で出す
- 権限分離: 閲覧だけのアクションと更新可能なアクションでAPIトークンを分ける
- 実行履歴: 実行者、実行モード、件数、エラー内容を `同期ログ` に残す
- 行単位エラー: 失敗行に `同期エラー` を書き戻し、成功行だけ `同期済み` にする
- 小さく開始: まずは1アプリ、1シート、5件程度でテストしてから対象を広げる
- 削除保留: 初期版では削除を自動同期せず、`削除候補` のような状態列で止める
- トークン非表示: APIトークンは設定シートとサーバー側だけで扱い、ブラウザへ返さない

## ローカル確認

このフォルダを静的配信すると、サンプル設定で画面操作を確認できます。

```bash
cd kintone-sheets-sync-app
python3 -m http.server 8787
```

ブラウザで `http://127.0.0.1:8787/` を開きます。
