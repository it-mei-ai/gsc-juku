# 学習塾向け 定期テスト現状分析アプリ 仕様書

作成日: 2026-07-18

## 1. アプリの目的

学校の定期テスト用紙を写真で取り込み、AIが正解・不正解・単元・ミス傾向を整理することで、生徒本人・保護者・学習塾の先生が「次に何を勉強すればよいか」をすぐ確認できるようにする。

模試のような偏差値・判定ではなく、定期テストでは見えにくい「どの単元でつまずいているか」「どんな間違い方をしているか」「塾で優先して復習すべきところはどこか」を可視化する。

## 2. 想定ユーザー

- 生徒: 自分の得意・苦手を把握し、次に取り組む単元を知りたい。
- 保護者: テスト結果を見ても、どこをサポートすればよいか分からない状態を解消したい。
- 塾の先生: 生徒ごとの弱点を短時間で把握し、授業・宿題・面談に活用したい。
- 管理者: 生徒・教科・テスト結果を管理し、アプリの運用状況を確認したい。

## 3. MVPで解決する課題

定期テストは点数と順位だけが返ってくることが多く、本人も保護者も「何ができていないのか」まで分析しにくい。

MVPでは、以下を最小機能として実現する。

1. Googleドライブ上のテスト写真をアプリに取り込む。
2. 写真から問題・解答・赤丸/バツ/部分点/コメントをAIで読み取る。
3. 教科ごとに、正解・不正解・単元・ミス原因を整理する。
4. 強み・弱み・優先復習ポイントを日本語で表示する。
5. 塾の先生に見せられる「次回学習メモ」を出す。

## 4. 初期スコープ

### 対象教科

初期対応は以下の2教科を優先する。

- 数学
- 英語

追加候補:

- 国語
- 理科
- 社会

### 対象資料

- 学校の定期テスト用紙の写真
- 採点済み答案の写真
- 解答用紙の写真
- 必要に応じて、問題用紙と答案用紙の両方

補足:

- 写真だけで正答が判断できない場合があるため、可能であれば「採点済み答案」を必須にする。
- 解答や配点が写真に写っていない場合、AIは「推定」として扱い、確信度を下げる。
- 生徒名など個人情報はアップロード前に隠す運用を推奨する。

## 5. 主要機能

### 5.1 Googleドライブ連携

ユーザーはGoogleアカウントでログインし、テスト写真を保存しているGoogleドライブのフォルダを選択する。

機能:

- Googleログイン
- Google Drive Pickerでフォルダ選択
- 選択フォルダ内の画像/PDF一覧取得
- 新規ファイルの検出
- 取り込み済み/未解析/解析済み/エラーの状態管理

MVPでは「手動同期」ボタンでフォルダを再読み込みする。

正式版では、定期実行またはDrive変更通知で自動更新する。

### 5.2 テスト登録

テスト写真に対して以下の情報を設定する。

- 生徒
- 学年
- 教科
- テスト名
- 実施日
- 学校名またはテスト範囲
- 写真ファイル

MVPでは、写真選択後に教科・テスト名を手入力する。

正式版では、AIが写真から教科・単元・点数を推定し、ユーザーが確認する。

### 5.3 AI読み取り

AIが画像から以下を抽出する。

- 問題番号
- 問題文の概要
- 生徒の解答
- 採点結果
- 配点/得点
- 赤丸/バツ/三角/部分点
- 先生のコメント
- 単元
- ミス原因
- 確信度

AIの出力は必ず構造化データとして保存する。

例:

```json
{
  "questionNo": "大問2 (3)",
  "subject": "数学",
  "unit": "二次方程式",
  "topic": "因数分解を使った解法",
  "score": 0,
  "maxScore": 4,
  "result": "incorrect",
  "mistakeType": "計算手順の理解不足",
  "comment": "式変形の途中で符号を取り違えている可能性が高い",
  "confidence": 0.78
}
```

### 5.4 現状分析

解析結果をもとに、教科別に以下を表示する。

- 総得点/満点
- 正答率
- 単元別正答率
- よく間違える単元
- 安定して得点できている単元
- ミスの種類
- 次に復習すべき優先順位

数学の分析例:

- 二次方程式の計算で失点が多い
- 符号ミス・移項ミスが目立つ
- 図形の基本問題は正答できている
- 応用問題よりも、まず計算手順の確認が必要

英語の分析例:

- 時制の選択で間違いが多い
- 三単現のs、過去形、疑問文の語順でつまずいている
- 単語の意味理解は比較的安定している
- 文法問題の反復演習が優先

### 5.5 強み・弱みレポート

生徒向け・保護者向け・先生向けに、同じ分析結果を異なる表現で表示する。

生徒向け:

- よくできているところ
- 次に伸ばせるところ
- 今日からやること

保護者向け:

- 現状の見立て
- 家で声かけするとよいポイント
- 塾で相談するとよいこと

先生向け:

- 授業で優先すべき単元
- 宿題候補
- 面談時に伝える要点

### 5.6 次回学習メモ

塾の先生にそのまま見せられる短いメモを生成する。

例:

```text
数学は二次方程式の計算、特に符号と移項で失点が出ています。
まずは基本計算を10問ほど確認し、途中式を書く練習を優先したいです。
図形の基礎問題は安定しているので、ここは維持しながら応用へ進めます。
```

### 5.7 履歴・グラフ

生徒ごとに過去の分析結果を蓄積する。

表示:

- 教科別の点数推移
- 単元別の正答率推移
- 苦手単元の改善状況
- 直近テストの要約
- 過去のテスト写真へのリンク

MVPでは1人分の履歴表示を優先し、複数生徒管理は正式版で強化する。

## 8. AI解析仕様

### 入力

- テスト写真
- 教科
- 学年
- テスト名
- テスト範囲
- 必要に応じて、ユーザーの補足メモ

### 出力

- 構造化JSON
- 生徒向け自然文
- 保護者向け自然文
- 先生向け自然文

### 解析ルール

- 採点済みの赤丸/バツを優先して正誤判断する。
- 正答が写真内にない場合は、断定せず「推定」として扱う。
- 単元分類は中学生向けの粒度にする。
- 点数よりも「次に何をやるか」が分かるコメントを重視する。
- 苦手だけでなく、得意・伸ばせる分野も必ず出す。
- AIの確信度が低い箇所は、ユーザー確認を促す。

### ミス分類

数学:

- 計算ミス
- 符号ミス
- 公式理解不足
- 問題文の読み取り不足
- 途中式不足
- 応用問題の方針不明

英語:

- 単語
- 熟語
- 文法
- 時制
- 三単現
- 語順
- スペル
- 長文読解
- 英作文

## 9. 権限・個人情報

公開URLでアクセスできるアプリにするが、生徒データは公開しない。

必須方針:

- メールアドレス + 生徒ID / 先生IDによるマジックリンクログイン
- ユーザーごとに見える生徒/テストを制限
- Driveの元画像はユーザー本人のGoogleドライブに保持
- アプリ側には解析結果とDriveファイルIDを保存
- 個人名・学校名・答案画像の取り扱いに注意
- デモでは匿名化済みサンプルを使う

保護者・生徒向けに、以下を明示する。

- 何のために写真を読み取るか
- どのデータを保存するか
- AI解析には誤りがあり得ること
- 最終判断は先生または保護者が確認すること

### 9.1 ログイン仕様

ログインは、パスワードを使わないマジックリンク方式にする。

事前登録データ:

- 生徒ID
- 生徒名
- 学年
- 保護者メールアドレス
- 生徒メールアドレス
- 担当先生ID
- 先生メールアドレス
- 閲覧可能な生徒ID

ログイン手順:

1. ユーザーがメールアドレスと生徒ID / 先生IDを入力する。
2. サーバーが事前登録済みの組み合わせと照合する。
3. 一致した場合だけ、メール宛にログインURLを送る。
4. ログインURLは15分程度で失効する。
5. ログインURLは1回使ったら無効化する。
6. ログイン後も、APIアクセスごとに閲覧権限を確認する。

セキュリティ方針:

- 生徒IDは連番ではなく、推測しにくいランダム文字列にする。
- URLには生徒IDやメールアドレスを入れず、ランダムな一時トークンだけを入れる。
- エラーメッセージでは、メールアドレスや生徒IDの存在有無を明かさない。
- 同じメールアドレスやIPからの連続試行には回数制限を入れる。
- 保護者/生徒は自分に紐づく生徒だけ閲覧できる。
- 先生は担当生徒だけ閲覧できる。
- 管理者だけが全生徒を管理できる。

## 10. デプロイ方針

公開して使えるWebアプリとしてデプロイする。

推奨構成:

- フロントエンド/バックエンド: Next.js
- 認証: Google OAuth
- Drive連携: Google Drive API + Google Picker
- データベース: Supabase Postgres または Firebase
- AI解析: 画像対応AI API
- デプロイ: Vercel / Firebase Hosting / OpenAI Sites のいずれか

MVPでは以下を優先する。

- 公開URLでアクセスできる
- Googleログインできる
- Driveフォルダを選べる
- 写真を解析できる
- 結果を保存できる

## 11. 開発フェーズ

### Phase 0: 仕様確定

- 対象ユーザーを確定
- MVP範囲を確定
- サンプル答案を用意
- 匿名化ルールを決める

### Phase 1: デモMVP

目的: 学習会や打ち合わせで見せられるデモを作る。

機能:

- サンプル画像のアップロード
- 教科選択
- AI解析
- 強み/弱み/次回学習メモ表示
- 1人分の履歴表示

この段階ではGoogle Drive連携を簡易化し、アプリ内アップロードでもよい。

### Phase 2: Drive連携MVP

目的: Googleドライブフォルダを選ぶと、テスト写真を読み込めるようにする。

機能:

- Googleログイン
- Driveフォルダ選択
- 手動同期
- 未解析ファイル一覧
- 解析結果保存

### Phase 3: 塾運用版

目的: 実際の塾で複数生徒を管理できるようにする。

機能:

- 生徒管理
- 保護者共有
- 先生向け一覧
- PDF出力
- 権限管理
- エラー確認

### Phase 4: 自動更新・高度分析

目的: 運用負担を下げ、長期的な学習支援に使えるようにする。

機能:

- Drive変更通知または定期同期
- 複数テストの傾向分析
- 学習計画自動生成
- 宿題候補
- 面談資料生成
- 教科追加

## 12. MVPの受け入れ基準

- テスト写真を1枚以上登録できる。
- 数学または英語を選択して解析できる。
- 正解/不正解/不明を問題ごとに表示できる。
- 単元とミス原因が表示される。
- 強みが1つ以上、弱みが1つ以上表示される。
- 次に学ぶべき単元が優先順位付きで表示される。
- 先生に見せる短いメモが生成される。
- 解析結果が保存され、後から見返せる。
- 公開URLからログインして利用できる。

想定デモ日: 2026-07-26(日) 18:00-21:00 の学習会

デモで伝えること:

- Google Driveは保護者個人のDriveか、塾側の共有Driveか。

## 15. 次に決めること

開発に入る前に、以下を決める。

1. 初回デモは「Drive連携あり」か「画像アップロードのみ」か。
2. サンプル答案は数学・英語それぞれ何枚用意できるか。
3. デモで見せる生徒は架空の1人にするか。
4. ログイン必須にするか、デモだけログインなしにするか。
5. デプロイ先をどこにするか。

推奨:

- 初回デモは、ログインなし・サンプル画像内蔵・解析結果表示までを最短で作る。
- その後、GoogleログインとDrive連携を追加する。
- 公開運用では必ずログインと権限管理を入れる。
